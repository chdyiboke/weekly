### 题目三
什么是事件代理？在dom 操作中的优势

是利用事件冒泡机制，将原本需要绑定在多个子元素上的事件，统一绑定到它们的父元素（或更上层的祖先元素）上。当子元素触发事件时，事件会向上冒泡到父元素，父元素再通过event.target判断是哪个子元素触发了事件，进而执行对应的逻辑

优势
减少内存占用
不需要给每个子元素单独绑定事件，只需要给父元素绑定一次，避免了大量事件绑定带来的内存消耗。
动态元素适配
新增或删除子元素时，不需要重新绑定事件，父元素的事件处理逻辑会自动对新元素生效。
简化代码逻辑
避免了重复编写事件绑定的代码，让代码更简洁、易维护

总结：事件代理是利用事件冒泡，将子元素的事件绑定到父元素上的技巧；优势是减少内存占用、支持动态元素、简化代码逻辑

#### 找出无序数组中第二大的数
方法 1：一次遍历（最优，时间复杂度 O (n)）

```javascript
function findSecondMax(arr) {
  if (arr.length < 2) return null; // 边界处理：数组长度不足2
  let max = -Infinity, secondMax = -Infinity;
  for (const num of arr) {
    if (num > max) {
      secondMax = max;
      max = num;
    } else if (num > secondMax && num !== max) { // 排除与最大值相等的情况
      secondMax = num;
    }
  }
  return secondMax === -Infinity ? null : secondMax; // 处理全为相同元素的情况
}

// 示例
console.log(findSecondMax([3, 1, 4, 1, 5, 9, 2, 6])); // 6
console.log(findSecondMax([5, 5, 5])); // null

```
方法 2：排序后取（简单直观，时间复杂度 O (n log n)）

```javascript
function findSecondMaxBySort(arr) {
  if (arr.length < 2) return null;
  const uniqueArr = [...new Set(arr)]; // 去重
  if (uniqueArr.length < 2) return null;
  return uniqueArr.sort((a, b) => b - a)[1]; // 降序排序后取第2个
}
```

#### 字符串反转函数
```javascript
function reverseString(str) {
  return str.split('').reverse().join('');
}
```
递归实现（加分项）
```javascript
function reverseString(str) {
  if (str.length <= 1) {
    return str;
  }
  return reverseString(str.slice(1)) + str[0];
}
```

### 题目十五

从输入 URL 到页面完全渲染的过程
网络请求阶段
URL 解析：解析协议、域名、路径等信息，验证合法性。
DNS 查询：将域名解析为 IP 地址，依次查询浏览器缓存→系统缓存→路由器缓存→ISP DNS→根域名服务器。
建立连接：TCP 三次握手（HTTPS 需额外 TLS 握手）。
发送请求：构造 HTTP 请求报文，发送至服务器。
接收响应：服务器处理后返回 HTML 文件，浏览器接收数据。
浏览器渲染阶段
HTML 解析与 DOM 树构建：逐行解析 HTML，生成 DOM 节点树，JS 会阻塞解析与渲染，CSS 会阻塞渲染。
CSS 解析与 CSSOM 树构建：解析样式表，生成包含样式规则的 CSSOM 树。
渲染树构建：合并 DOM 与 CSSOM，生成仅包含可见节点的渲染树。
布局（Reflow）：计算节点的位置、尺寸等几何信息。
绘制（Paint）：将样式信息绘制为像素数据。
合成（Composite）：合并绘制层，输出到屏幕完成渲染。
